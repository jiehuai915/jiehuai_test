Industry Status

当前行业现状

企业 AI 正从“员工旁边的 Copilot”转向“系统里的数字劳动力”。胜负不只取决于模型能力，而取决于数据、权限、流程、审计和系统回写能否闭环。

已核验事实

  • Salesforce 在 2025 年推出 AgentExchange，官方称其扩展 Agentforce 生态，并把 Agentforce 定位为 digital labor platform。
  • Salesforce Agentforce 3 官方材料把规模化 AI agents 的最大阻碍定义为 visibility 与 control，并强调 unified agents、data、apps 与 metadata。
  • ServiceNow 2025 年发布新的 AI Platform，提出让 any AI、any agent、any model 在企业中协同工作；其 AI Agent Fabric 让 agents 和 orchestrators 共享上下文、协调任务并推动结果。
  • ServiceNow Yokohama release 宣布 AI Agent Orchestrator 与 AI Agent Studio GA，并扩展 Knowledge Graph 与 Common Service Data Model，以打通数据源壁垒。
  • Microsoft Research 开源 项目，把 LLM 生成的知识图谱与图机器学习结合，用于提升私有数据集上的问答质量。
  • Databricks 社区 2026 年发布 Agent Bricks 新能力，定位为 governed enterprise agent platform，让 AI agents 在业务数据上运行，并带有权限和控制。

一句话现状

企业 AI 的主战场已经从“模型 API 调用”转向 连接、/ 权限、/、 和可量化业务结果。

Topology

经典拓扑流向

企业落地链本质是一条“从脏数据到可信动作”的管道：先把数据和权限整理好，再让模型在边界内执行，并把结果写回业务系统。

企业数据 数据治理 RAG 权限系统 行业模型 Agent 业务系统

1. 企业遗留数据

数据分散在 CRM、ERP、ITSM、工单、邮箱、文档、BI、代码库和数据库里。真正难点是 、口径一致、权限继承和历史脏数据，而不是简单上传 PDF。

2. 数据治理 / 向量化

企业需要 、脱敏、标签、权限映射、、向量索引和审计。没有治理层，RAG 会把错误、过期或越权信息放大。

3. RAG / GraphRAG

把检索结果带入上下文， 用实体、关系和社区摘要提高复杂知识库问答质量。它们解决“模型不知道企业私有事实”的问题，但不自动解决权限和质量。

4. 权限系统

企业 AI 必须继承原系统权限：谁能看客户报价、谁能改订单、谁能查员工信息。、、 脱敏和 是上线门槛。

5. 行业模型 / 混合路由

企业不会只用一个模型。常见架构是大模型处理复杂推理，小模型处理分类和抽取，私有模型处理敏感数据， 根据任务、成本和合规要求分流。

6. 多 Agent 工作流

生产级 Agent 需要计划、工具调用、状态管理、审批、回滚和审计。 的价值是把多个专用 agent 编排成可控流程，而不是让模型自由行动。

7. ERP / CRM / ITSM 回写

价值最终发生在系统回写：创建报价、更新工单、生成采购申请、触发审批、修改客户记录。没有 回写，AI 只是问答工具，不是生产系统。

Interface Friction

2026 年连接处核心痛点

数据与权限接口

  • 权限继承难：AI 检索结果必须遵守原系统权限，否则 RAG 会变成越权泄露通道。
  • 数据质量不稳定：企业文档过期、字段口径冲突、工单噪声、Excel 私有口径会让模型“认真地错”。
  • 多租户隔离：SaaS 平台要同时做缓存复用、成本优化和客户数据隔离，工程上很难两全。
  • 合规与数据驻留：金融、医疗、政企客户会要求私有部署、区域隔离、审计日志和供应商责任边界。

工作流与 ROI 接口

  • 幻觉率和责任边界：企业最怕 AI 自信地写错系统记录，因此关键动作需要 HITL、阈值和回滚。
  • 系统集成 TCO：License 只是小头，连接器、权限映射、数据清洗、流程改造和运维才是大头。
  • 早期 ROI 低：很多项目停在试点，因为没有把 KPI 绑定到工单关闭率、销售转化、客服 AHT、财务关账周期等指标。
  • 组织采纳：AI 改变岗位边界，业务部门、IT、安全、法务和财务之间需要新的治理机制。

CAPEX & Value Capture

长期价值捕获与资本开支变量

1. 系统入口与数据所有权

Salesforce、ServiceNow、Microsoft、SAP、Workday 的护城河来自系统入口、元数据、权限模型和业务对象。AI 越深入流程，越依赖这些 system of record。

2. 数据平台与治理层

Databricks、Snowflake、Microsoft Fabric、Google BigQuery 等捕获企业数据整理、权限、语义层、向量化和模型开发预算。

3. 连接器与集成服务

MuleSoft、Boomi、Workato、Zapier、n8n、iPaaS 和咨询集成商赚的是系统碎片化的钱。连接越深，切换成本越高。

4. Agent 编排与治理

Agent Studio、Agent Orchestrator、LangGraph、CrewAI、企业 sandbox 和 observability 工具捕获生产级 agent 的可控性预算。

5. 行业数据与专业工作流

医疗、法律、金融、制造、客服等行业软件公司拥有语料、流程和监管理解，可以把 AI 从通用助手变成垂直 autopilot。

6. 安全、审计与合规

身份治理、DLP、审计日志、prompt 防火墙、数据驻留和供应商风险管理会成为企业 AI 扩张的硬预算，而不是附加功能。
