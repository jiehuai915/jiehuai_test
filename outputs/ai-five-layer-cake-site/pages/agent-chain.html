Industry Status

当前行业现状

Agent 已从“提示词玩法”进入平台化阶段：模型厂商提供 tool use 和 tracing，协议层出现 MCP，工程框架开始强调 durable execution、human-in-the-loop 和生产观测。

已核验事实

  • OpenAI 2025 年发布 Responses API、内置工具和 Agents SDK，并说明 Responses API 是构建 agents 的新 API primitive，Agents SDK 支持 tracing。
  • OpenAI Agents SDK 官方文档将 agents 定义为可使用额外上下文和工具、可 handoff 给专门 agents、可流式输出并保留完整 trace 的 agentic application 构建库。
  • Anthropic 2024 年开源 ，用于把 AI assistants 连接到数据、业务工具和开发环境；2025 年 Anthropic 将 MCP 捐给 Linux Foundation 的 Agentic AI Foundation。
  • MCP 官方 tools 文档说明 MCP 允许 servers 暴露可被语言模型调用的工具，这是 Agent 工具层标准化的关键。
  • LangGraph 官方文档把自己定位为 low-level orchestration framework，支持 durable execution、human-in-the-loop、memory 和 production agent 部署。
  • 2026 年 MCP 安全事件和研究讨论显示，Agent 工具连接标准化会带来新的供应链和权限风险，企业需要 sandbox、allowlist 和运行时审计。

一句话现状

Agent 的商业化关键不是“自动点几个工具”，而是 能否把任务拆解、状态记忆、工具选择、权限边界、失败重试、人类审批和审计日志稳定地组织起来。

Topology

经典拓扑流向

Agent 链不是一个模型，而是一套运行时：模型负责推理，runtime 负责状态、工具、权限、回滚和人类协作。

基础模型 Planning Memory Tools/MCP Sandbox HITL Audit

1. 基础模型

Agent 的能力上限来自 、tool calling、多模态理解和长上下文。模型越强，规划和工具使用越可靠，但成本和延迟也会上升。

2. 目标拆解与规划

把用户目标拆成步骤、依赖、工具调用和验收条件。生产环境常用 workflow graph、state machine 或 planner/executor 架构限制 Agent 任意发挥。

3. 长期 / 短期记忆

包括短期上下文、长期用户偏好、项目状态、向量检索和结构化状态。核心难点是记得有用信息，同时避免污染、过期和越权。

4. 工具调用与 API 连接

、function calling、API connectors、database tools 和 browser/computer use 让 Agent 能行动。工具 schema、权限、输入校验和结果验证决定执行质量。

5. Sandbox / 隔离执行

把代码执行、文件操作、浏览器控制和第三方 API 调用限制在隔离环境中。Agent 越能行动，sandbox 和权限越重要。

6. 多 Agent 协同 / 人类在环

复杂任务常由 planner、researcher、coder、reviewer、operator 等专用 agent 协作。高风险动作需要 审批、阈值控制和可回滚机制。

7. Trace / Audit / 评测

记录每一步输入、工具调用、模型输出、审批和错误。没有 trace，就无法调试、归责、评测或满足企业合规。

Interface Friction

2026 年连接处核心痛点

模型到工具接口

  • 长上下文衰减：上下文越长，模型越容易漏看关键约束；工具列表过多也会挤占注意力和 token 预算。
  • 工具选择错误：Agent 调错 API、传错参数、误解返回值，都会把语言错误变成业务系统错误。
  • Looping / 死循环：Agent 在规划、检索、修复、重试之间循环，烧 token 但不推进任务。
  • 非确定性：同一目标多次运行可能走不同路径，企业难以保证 SLA、成本和责任边界。

行动到治理接口

  • 权限边界：读取、写入、支付、发送、删除、部署等动作风险不同，必须分级授权。
  • MCP 供应链风险：工具 server、SDK、stdio 本地进程和第三方连接器都可能成为攻击面。
  • 状态恢复：长任务会跨分钟、小时甚至天运行，runtime 必须支持 checkpoint、resume 和 idempotency。
  • 可解释审计：企业不只要结果，还要知道 Agent 为什么这么做、调用了什么、谁批准了什么。

CAPEX & Value Capture

长期价值捕获与资本开支变量

1. Agent Runtime

Agent runtime 负责状态、工具、记忆、权限、重试、trace 和部署。它是把模型 API 变成生产系统的中间层。

2. 工具协议与连接器

MCP、function calling、iPaaS、SaaS connectors 和企业 API 网关捕获“让 Agent 接入业务系统”的预算。

3. 记忆与知识层

向量库、GraphRAG、项目状态存储、用户偏好和会话摘要决定 Agent 能否长期工作，而不是每次从零开始。

4. 安全与权限控制

企业愿意为 allowlist、sandbox、secret vault、DLP、approval workflow、audit log 和 policy engine 付费。

5. 评测与可观测性

LangSmith、OpenAI tracing、Arize、Helicone、Datadog 等捕获 Agent 运行质量、成本和失败复盘预算。

6. 垂直 Agent 应用

代码、销售、客服、IT 运维、财务、法律和数据分析 Agent 最容易形成专有工作流和数据闭环。
