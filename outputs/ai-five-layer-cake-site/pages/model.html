Fundamentals

经典基础知识筑基

概率建模与最大似然

本质上学习数据分布，在给定上下文下预测下一个 token。模型能力来自数据分布、参数容量、优化过程和后训练反馈的共同作用。

与注意力机制

注意力机制让模型在上下文内动态建立 token 关系，但上下文越长，计算与 成本越高。

预训练、后训练与对齐

学世界知识与语言结构， 学任务格式，/ 调整偏好与安全边界。

评测与泛化

模型上线不是只看 benchmark 分数，而要看真实任务成功率、幻觉率、工具调用可靠性、安全性、成本和延迟。

前沿技术路线

o1/o3 类

模型在回答前消耗更多推理计算进行内部规划、验证和工具调用。能力提升的代价是 爆炸和推理成本上升。

分离与 工程

长上下文读入和逐 token 生成是两类负载。推理系统通过解耦、缓存、路由和批处理优化 、吞吐与成本。

原生多模态模型

文本、图像、语音、视频、屏幕、空间和动作轨迹逐步进入统一模型，推动搜索、办公、编程、机器人和自动驾驶应用重构。

与物理 AI

Genie、Gemini Robotics、GR00T 等路线把模型从语言空间推向可预测、可交互的物理世界表征。

小模型、蒸馏与路由

大模型做 teacher，小模型做高频执行， 在能力、成本、延迟之间动态分配任务。

合成数据与自我改进

能扩充稀缺任务数据，但会带来数据污染、模式坍塌和系统性偏见。

Bottlenecks

卡脖子瓶颈

能力与成本瓶颈

  • Reasoning token 爆炸：长时思考模型提高正确率，同时把单位任务成本推高。
  • ：上下文窗口增大不等于可无限有效使用，注意力、缓存和检索质量都会退化。
  • ：长上下文、多轮对话和多用户并发的显存黑洞。
  • 高质量数据枯竭：公开互联网文本边际质量下降，私有数据与合成数据成为竞争焦点。

可靠性与商业化瓶颈

  • ：企业落地最直接的信任成本。
  • 工具调用可靠性：模型说对不等于能安全执行，Agent 需要可控状态机和权限边界。
  • 评测污染：公开 benchmark 容易被训练数据污染，真实任务评测更重要。
  • 安全与版权：数据来源、输出责任、隐藏思考链和合规风险仍在快速变化。

Interface Friction

模型层与上下游的接口摩擦

模型层的摩擦来自“能力提升”和“生产可用”之间的差距：更强模型常常同时带来更高成本、更高延迟和更难解释的行为。

模型能力 vs 推理成本

Reasoning、长上下文和多模态提升任务完成率，但会推高 、 占用、P99 和单任务 token 成本。

通用模型 vs 企业私有数据

企业要的是基于权限、来源和业务语义的可信答案。模型必须通过 、GraphRAG、微调和评测接入企业数据，不能只靠通用能力。

Benchmark 高分 vs 真实任务成功

公开评测容易被污染或过拟合，真实业务看 Agent Success Rate、幻觉率、工具调用成功率、可回滚性和人工介入比例。

隐藏思考链 vs 安全审计

推理模型的内部思考不一定完全暴露，给 jailbreak、越权工具调用和事故归因带来新难题。安全评测必须从输出审查升级到行为审计。

CAPEX / Value Capture

资本开支与价值捕获

模型层的资本开支正在从一次性训练军备竞赛，转向训练、后训练、推理优化、评测和数据飞轮的长期复合投入。

核心 CAPEX / OPEX 变量

  • 训练算力：GPU/HBM、网络、存储、数据管线和失败重跑成本。
  • 后训练与数据：SFT、RLHF/RLAIF、人类反馈、合成数据和领域评测集。
  • 推理 OPEX：reasoning token、上下文长度、batching、cache 命中率和区域部署。
  • 安全评测：红队、动态 benchmark、运行时监控、内容安全和事故响应。

价值捕获位置

  • 前沿模型 API：靠能力领先、开发者生态和企业信任获得溢价。
  • 高效小模型/路由：用成本优势吃掉高频任务，保护应用毛利。
  • 专有数据飞轮：行业数据、用户反馈和任务结果形成差异化能力。
  • 模型运维与评测：企业上线需要持续 eval、guardrails、监控和合规报告。
