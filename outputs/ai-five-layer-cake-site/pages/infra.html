Layer 03 · Infra

基础设施层：把 GPU 变成可售卖的 AI 工厂

核心矛盾：资本市场看到的是多少张卡，产业真正比拼的是这些卡能否在电力、网络、液冷、调度和推理服务中持续产出高利用率 token。

当前判断 基础设施层正在从传统云计算升级为 AI Factory：训练要解决万卡并行效率，推理要解决 prefill/decode、KV cache、P99 延迟和单位 token 成本。

Industry Status

当前行业现状

已核验事实

  • NVIDIA 的 机柜级系统连接 36 个 Grace CPU 和 72 个 Blackwell GPU，官方称其以液冷 rack-scale 设计服务万亿参数模型。
  • NVIDIA 宣布 Ethernet 能把多个分布式数据中心连接成 gigascale AI super-factory。
  • NVIDIA 是面向 reasoning 模型和推理服务的开放式分布式运行时，核心方向包括 解耦和 优化。
  • Meta 官方披露其两个 24,576 GPU 集群用于 Llama 3 训练，并把网络、存储、功耗和硬件故障作为大规模训练关键问题。
  • xAI 官方称 Colossus 是 100K Hopper GPU 超级计算机，并已扩展至 200K GPU 规模，凸显 AI infra 的军备竞赛。
  • OpenAI 宣布 Stargate 计划，未来四年拟投资 5000 亿美元建设美国 AI 基础设施，初始部署 1000 亿美元。

一句话现状

AI 基础设施的竞争已从“云上有没有 GPU”变成“谁能以更低故障率、更低延迟、更高 和更低单位 token 成本，把训练集群与推理集群工业化运营”。

Interface Friction

基础设施层与上下游的接口摩擦

基础设施层是五层蛋糕里最容易“纸面算力虚高”的地方：采购 GPU 不等于获得稳定、可售、可计费的 AI 产能。

电力交付 vs GPU 到货

GPU 服务器到货以后，还要等待机房电力、液冷、母线、CDU 和网络联调。AI 工厂的产能爬坡往往被设施侧节奏拖住。

训练网络 vs 推理网络

训练追求 效率，推理追求 、P99 和高并发隔离。一个集群很难同时对两类负载都最优。

GPU 利用率 vs 企业 SLA

更 aggressive 的 batching 可以提高吞吐和利用率，但会伤害尾延迟。企业 SLA 要的是稳定 P99，不是平均成本最低。

云平台抽象 vs 物理故障

云客户看到的是实例和 API，底层真实面对的是光模块故障、网卡抖动、液冷告警、checkpoint I/O 和调度碎片化。

Optical / PCB Fabric

光互联、光模块、光芯片与高速 PCB

AI 基础设施的网络瓶颈正在从“交换机有没有”升级为“每 bit 的功耗、误码率、良率、布线半径和信号完整性”。这条链是训练 MFU、推理 P99 和 AI 工厂交付节奏的共同底座。

800G / 1.6T

400G/800G 之后，1.6T 成为 AI 后端网络升级的核心迁移方向。模块厂的竞争不只看出货，还要看 功耗、激光器良率、客户认证和散热。

/

当交换芯片速率继续上探，传统可插拔光模块的功耗和面板密度会受压。CPO 与硅光尝试把光互联更靠近交换芯片，但运维、可替换性和封装良率仍是工程摩擦。

/ 高速铜缆

在 rack 内短距连接里，有源铜缆和 retimer/SerDes 可以用更低成本承接部分光模块需求。它是光模块与 CPO 之间的过渡路线，也会影响 Credo、Amphenol、Astera 等连接链价值。

/ / 背板

112G/224G 会把信号完整性压力传导到低损耗 CCL、高多层 PCB、背钻、阻抗控制和连接器。AI 服务器、交换机和背板的材料等级开始成为网络性能瓶颈。

代表标的

  • 光模块/光器件：Coherent、Lumentum、Fabrinet、中际旭创、新易盛、光迅科技。
  • SerDes/DSP/连接：Broadcom、Marvell、Credo、Astera Labs、Amphenol。
  • PCB/CCL/背板：生益科技、沪电股份、深南电路、Unimicron、Ibiden、Shinko。

核心 KPI

  • 光模块速率：400G/800G/1.6T，决定带宽密度与模块价值量。
  • 每 bit 功耗：网络功耗会进入 AI 数据中心 PUE 与 rack thermal budget。
  • SerDes lane rate：112G/224G 决定交换芯片、PCB、连接器和 DSP 难度。
  • PCB 插入损耗：低损耗材料、层数、背钻和阻抗控制决定高速信号可用性。

CAPEX / Value Capture

资本开支与价值捕获

基础设施层把硬件资本开支转化为可售 token 和模型能力，真正的护城河来自利用率、可靠性、交付速度和客户负载绑定。

核心 CAPEX 变量

  • GPU/整柜采购：NVL72、HGX、网络、存储和机柜级系统共同决定单 MW 算力产出。
  • 网络与光互联：InfiniBand、Spectrum-X、Ultra Ethernet、光模块和交换机是训练效率的放大器。
  • 液冷与电力设施：高密 rack 把 CDU、冷板、UPS、母线和水处理纳入 IT capex。
  • 调度与可观测性：降低空转、故障恢复和多租户碎片化，决定同样硬件能卖出多少有效 token。

价值捕获位置

  • 超大云与 GPU 云：靠规模采购、电力卡位、客户合约和平台服务赚取算力租金。
  • 网络/存储/服务器链：AI 集群越大，非 GPU 设备价值量越不可忽视。
  • 推理 runtime：Prefill/decode、KV cache、路由、batching 直接改善毛利。
  • 运维自动化：MTBF/MTTR、checkpoint、故障隔离和容量规划构成长期经验曲线。
